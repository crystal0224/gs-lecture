# GS 강의 프레젠테이션 — 디자인 규칙

## 슬라이드 구조 원칙

### 제목
- 제목(h2)은 **한 줄**을 기본으로 한다. `<br>` 줄바꿈은 최소화.
- 제목은 **상단 중앙**에 위치. 현재 폰트 크기(clamp 34-54px) 유지.

### 레이아웃 선택 기준
- **`.ta` (top-anchored)**: 제목 상단 + 콘텐츠 확장형. 카드나 그리드가 메인인 슬라이드.
- **좌우 분할 (grid 1fr 1fr)**: 좌측 = 핵심 메시지 (수직·수평 중앙 정렬), 우측 = 상세 카드.
- **`.c` (center)**: 짧은 메시지, 인용, 전환 슬라이드에만 사용.

### 콘텐츠 영역
- `max-width`는 **1100px 이상**. 화면을 충분히 사용할 것.
- 카드는 `box-shadow` + `backdrop-filter: blur(8px)`로 **입체감** 부여.
- 카드 내 서술은 **구체적이고 직관적**으로. 추상적 키워드 나열 금지.

## 요소 구분: 카드 vs 메시지

### 카드에 넣는 것 (정보)
- 데이터, 팩트, 상황 설명, 비교 항목
- 구조화된 정보 (→ AI / → 사람 같은 분류)
- `border`, `background`, `border-radius` 있음

### 텍스트로 빼는 것 (메시지)
- Takeaway, 결론, 감정적 울림이 있는 한 줄
- 하단 또는 카드 아래에 `<p>` 태그로 배치
- 카드로 감싸지 않음. 강조는 `color: var(--copper)` 또는 `font-weight: 700`

## 카드 내부 구조

### 기본 패턴
```
제목 (굵게, white)
— 구체적 설명 (g500, 대시 구분자)
```

### 비교형 패턴 (vs-side)
```
레이블 (copper 또는 teal, uppercase)
제목 (white)
— 설명 1
— 설명 2
```

### 분류형 패턴 (AI/사람 구분)
```
→ AI   설명 (teal)
→ 사람  설명 (copper)
기대효과 — 한 줄 (g400, 상단 border-top 구분)
```

## 색상 사용 규칙
- **copper (#D4A574)**: 강조, 경고, 핵심 키워드, 홀수 번호
- **teal (#4DC9C4)**: 보조 강조, 데이터, 짝수 번호
- **white**: 제목, 핵심 메시지
- **g400~g500**: 본문 설명
- **g300**: 대시(—) 구분자, 비활성 텍스트

## 발표자 노트 (data-n)
- 슬라이드에 담지 않는 구술 에피소드를 넣는다.
- SK 경험, 현장 사례는 data-n에 자연스럽게 녹인다.
- 장표에 "SK 사례입니다"라고 명시하지 않는다.

## 이미지 톤앤매너

슬라이드에 삽입되는 이미지는 아래 기준을 통일한다.

| 항목 | 지정값 |
|------|--------|
| 배경 | #0A0A0A (프레젠테이션과 통일) |
| 강조색 1 | copper/amber #D4A574 |
| 강조색 2 | teal #4DC9C4 |
| 화풍 | editorial illustration, minimal, metaphorical |
| 질감 | slight film grain, clean lines |
| 레퍼런스 | Bloomberg Businessweek, The Economist 표지 스타일 |
| 금지 | 텍스트, 사람 얼굴, 사진 느낌, 과한 채도 |

이미지 생성 프롬프트 작성 시 위 키워드를 기본으로 포함한다.
