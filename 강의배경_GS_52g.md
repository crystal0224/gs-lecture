# GS 52g 강의 배경

## 강의 정보

- **제목**: AI 시대, 일하는 방식의 재설계
- **발표자**: 배수정
- **대상**: GS 52g 크루
- **시간**: 50분 (발표 35분 + 토론/활동 15분)
- **상태**: 일정 미확정, 사전미팅 예정

---

## 52g (5pen 2nnovation GS) 개요

| 항목 | 내용 |
|------|------|
| **시작** | 2020년~ |
| **성격** | GS그룹 공통 혁신 프로그램 |
| **운영** | 각 계열사 + 지주사에서 선발된 크루가 **전업**으로 혁신 과제 기획·실행 |

### 크루 구성

| 구분 | 배경 | 역할 |
|------|------|------|
| **계열사 크루** | 인사, 재무, 생산, 연구 등 다양한 현업 | 각 계열사 내 현업 문제 발굴 → AI로 해결하는 프로젝트 PM |
| **지주사 크루** | 개발자, 디자이너 | 계열사 크루와 협업하여 과제 실행 지원 |

52g 측에서 발표자에게 요청한 관점:

3. **성공/실패 사례**

### 기존 강의 자료와의 접점

현재 강의안(LG HS본부 JB/CA 세션 기반)의 핵심 메시지:

| 강의 메시지 | 52g 맥락에서의 의미 |
|-------------|---------------------|
| AI가 개인을 강하게 만들지만 팀은 쪼갠다 | 계열사 크루(비개발)와 지주사 크루(개발자) 간 속도·기대 차이 |
| 저자의 실종 — 판단한 사람이 모호해진다 | AI로 만든 과제 기획서에서 "누구의 판단인가" 문제 |
| 학습의 증발 — 원리 없이 결과만 얻는다 | 크루가 AI 도구에 의존하면 과제 종료 후 역량이 남지 않을 수 있음 |
| 인센티브 부재 — 빨리 해봐야 일만 더 온다 | 전업 혁신이라 덜할 수 있으나, 계열사 복귀 후 인정 문제와 연결 |
| 말 한마디 바꾸기 / 팀의 약속 | 계열사-지주사 협업 그라운드룰로 직접 적용 가능 |

### 52g 특성상 커스터마이징 포인트

1. **청중이 "혁신 전담 크루"** — 일반 구성원이 아니라 이미 AI를 적극 활용 중인 사람들. "AI를 써야 한다"는 설득이 필요 없음. 오히려 **"잘 쓰는 사람이 만드는 팀 내 문제"**에 공감이 클 것
2. **계열사 크루 = PM, 지주사 크루 = 실행** — 이 구조 자체가 "속도 차이 → 기대 어긋남"의 현장. 강의의 "3개 축 어긋남" 슬라이드가 직접적으로 적용 가능
3. **전업 혁신이라 복귀 후 문제** — "학습의 증발"이 개인 차원뿐 아니라 조직 차원으로 확대 (크루 종료 후 계열사에 지식이 남는가?)
4. **성공/실패 사례 요청** — 발표자 본인의 경험(HR 시스템 프로젝트) + 추가 사례 필요. 52g과 유사한 "혁신 조직"의 AI 도입 사례가 좋을 것
